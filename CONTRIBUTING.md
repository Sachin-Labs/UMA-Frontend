# Contributing to UMA Frontend

We love your input! We want to make contributing to UMA as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features

## 🚀 Simple Steps to Contribute

1.  **Fork** the repository to your own GitHub account.
2.  **Clone** your fork to your local machine.
3.  **Create a branch** for your changes (see naming conventions below).
4.  **Commit** your changes with a clear and descriptive message.
5.  **Push** your branch to your fork.
6.  **Open a Pull Request** (PR) against the main repository.

## 🏷 Branch Naming Conventions

To keep things organized, please use the following prefixes for your branches:
- `feature/` : New UI components or functionality (e.g., `feature/login-animation`)
- `fix/`     : Bug fixes (e.g., `fix/sidebar-overlap`)
- `refactor/`: Code improvements without changing functionality
- `docs/`    : Documentation updates

## 🛠 Types of Contributions
We welcome contributions in all areas:
- **UI/UX**: Improving the design system, animations, or responsiveness.
- **Performance**: Optimizing bundle size or render cycles.
- **Bug Fixes**: Squashing visual glitches or logic errors.
- **Accessibility**: Making the app more usable for everyone.

## 📜 Basic Rules
- **Clean Code**: Follow the existing coding style and formatting.
- **Readable Commits**: Write meaningful commit messages (e.g., "Add loading state to login button" instead of "fixed stuff").
- **Test Your Changes**: Ensure your changes don't break existing functionality before submitting.
- **Be Respectful**: We are a community! Please follow our code of conduct.

Thank you for contributing to UMA! ❤️
